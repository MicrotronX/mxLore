           Self-Update
        

        

          Check GitHub for a newer mxLore release. When an update is available,
          a banner appears at the top of every admin page.
        

        

          
          
        

        
          Triggers a fresh POST /api/self-update/recheck (60 s cooldown per session).
        
      

    


    
    

      

        
Connect URLs

        

          Configure how team members reach your mxLore server. Internal URL is for LAN access (direct binary).
          External URLs are for reverse-proxy deployments (e.g. IIS with HTTPS).
        


        

          
          

            
            
          

          
        


        

          
          

            
            
          

        


        

          
          

            
            
          

        


        

          
          
          Comma-separated IP addresses. Required for X-Forwarded-For handling behind IIS/nginx.
        


        

          
          
        

      

    


    
    

      

        

           Syslog Viewer
        

        

          Coming soon — server log viewer with level filter (info / warn / error)
          and tail mode. Requires a dedicated backend endpoint
          (GET /api/logs?lines=N&levels=...) reading from
          logs/mxLoreMCP.log. Planned as Phase B of FR#3296.
        

        
          Until this ships, use the structured call-log via mx_briefing
          or look at logs/ on the server host directly.
